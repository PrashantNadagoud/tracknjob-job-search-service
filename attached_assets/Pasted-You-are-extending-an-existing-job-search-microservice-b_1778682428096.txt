You are extending an existing job search microservice built with 
FastAPI, PostgreSQL, Celery, Redis, and SQLAlchemy. The service 
crawls job listings from ATS platforms and stores them in 
jobs.job_listings with jobs.ats_sources tracking crawl sources.

Your task is to add India job coverage by:
1. Adding India-specific ATS sources for global companies 
   that post India jobs on their existing ATS platforms
2. Adding a location filter to Workday crawler to fetch 
   India-based jobs from existing sources
3. Seeding a verified list of Indian-native companies that 
   use standard ATS platforms

Group B — Indian-native companies that DO use standard ATS
A smaller but real set of companies confirmed on standard ATS.

Create scripts/validate_india_sources.py

For each row in jobs.ats_sources WHERE country='IN' 
AND is_active=false AND ats_slug IS NOT NULL:

Workday:
- POST to Workday jobs API with location_filter if set:
  URL: https://{slug}.wd{n}.myworkdayjobs.com/wday/cxs/{slug}/jobs
  Body: {"appliedFacets": {"Location": [location_filter]}, "limit": 5}
  Headers: User-Agent: Mozilla/5.0 (browser UA)
- Try wd1, wd3, wd5, wd12 in order
- If 200 + jobPostings present → is_active=true
- Update career_site_url with confirmed working URL

Greenhouse:
- GET https://boards-api.greenhouse.io/v1/boards/{slug}/jobs
- If 200 + jobs array present → is_active=true

Lever:
- GET https://api.lever.co/v0/postings/{slug}
- If 200 + items present → is_active=true

For all:
- 1 second delay between requests
- Update last_crawl_status: 'validated' or 'validation_failed'
- Update updated_at: now()

CLI:
python scripts/validate_india_sources.py
python scripts/validate_india_sources.py --limit 10
python scripts/validate_india_sources.py --dry-run

{

    "Location": [source.location_filter]
  },
  "limit": 20,
  "offset": 0
}

facet (existing behavior — do not change).

This ensures Accenture India only returns India-based 
postings, not all global Accenture jobs.
