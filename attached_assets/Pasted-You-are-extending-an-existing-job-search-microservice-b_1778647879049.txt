You are extending an existing job search microservice built with 
FastAPI, PostgreSQL, Celery, Redis, and SQLAlchemy. The service 
crawls job listings from ATS platforms (Workday, Greenhouse, Lever, 
SmartRecruiters) and stores them in a jobs.job_listings table.

Your task is to add a Job Alerts & Motivational Email Notification 
feature to this existing service.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
FEATURE OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Users subscribe to daily job alert emails. Every morning at 6 AM, 
the system:
1. Finds all active alert subscriptions
2. Queries job listings posted in the last 24 hours matching each 
   user's filters (title keywords, location, employment type)
3. If matches exist, generates a short AI motivational intro 
   paragraph using GPT-4o-mini (if enabled by user)
4. Sends a formatted HTML email via Resend API
5. Logs delivery status to DB

PART 1 — DATABASE SCHEMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Create a new Alembic migration that adds these two tables 
to the existing database:

Table: jobs.alert_subscriptions
- id: UUID primary key
- user_id: VARCHAR (external ID from TrackNJob frontend, not a FK)
- email: VARCHAR(255) NOT NULL
- name: VARCHAR(255) (for personalization)
- is_active: BOOLEAN DEFAULT true
- keywords: TEXT[] (array of job title keywords e.g. ["engineer", "python"])
- locations: TEXT[] (array of locations e.g. ["San Francisco", "Remote"])
- employment_types: TEXT[] (e.g. ["FULL_TIME", "CONTRACT"])
- ats_types: TEXT[] (filter by ATS, nullable = all)
- job_search_started_at: TIMESTAMP (when user started job hunting, 
  for "day X of your search" in email)
- motivational_email_enabled: BOOLEAN DEFAULT true
- delivery_time_utc: INTEGER DEFAULT 13 (1 PM UTC = 6 AM PDT)
- created_at: TIMESTAMP DEFAULT now()
- updated_at: TIMESTAMP DEFAULT now()

Table: jobs.alert_deliveries
- id: UUID primary key
- subscription_id: UUID FK → jobs.alert_subscriptions.id
- delivered_at: TIMESTAMP DEFAULT now()
- jobs_sent: INTEGER (count of job listings in this email)
- status: VARCHAR(50) ('sent', 'failed', 'skipped_no_matches')
- resend_message_id: VARCHAR (ID returned by Resend API)
- error_message: TEXT (if failed)

Create app/tasks/alert_tasks.py

Task: send_daily_alerts
- Runs via Celery Beat every hour (scheduler checks delivery_time_utc 
  per subscription, not a single global send time)
- For each active subscription where current UTC hour matches 
  delivery_time_utc AND no delivery exists for today:
  
  1. Query jobs.job_listings WHERE:
     - created_at >= now() - interval '24 hours'
     - title ILIKE ANY(keywords) [at least one keyword match]
     - location ILIKE ANY(locations) OR 'Remote' in locations
     - employment_type = ANY(employment_types) if set
     - ats_type = ANY(ats_types) if set
     ORDER BY created_at DESC
     LIMIT 10

  2. If 0 results → log alert_delivery with status='skipped_no_matches', skip

  3. If motivational_email_enabled AND OPENAI_API_KEY set:
     → call generate_motivational_intro(user_context)

  4. Render alert_email.html template with jobs + motivational text

  5. Send via Resend:
     from: "TrackNJob Alerts <alerts@tracknjob.com>"
     to: subscription.email
     subject: f"☀️ {jobs_found} new jobs for you today, {name}"

  6. Log to alert_deliveries:
     - status: 'sent' or 'failed'
     - jobs_sent: count
     - resend_message_id: from Resend response

Add to existing Celery Beat schedule:
'send-daily-alerts': {
    'task': 'app.tasks.alert_tasks.send_daily_alerts',
    'schedule': crontab(minute=0),  # runs every hour, task self-filters
}
