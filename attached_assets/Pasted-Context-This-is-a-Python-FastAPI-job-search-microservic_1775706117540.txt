Context
This is a Python/FastAPI job search microservice. The  CompanyEnricher  class lives in  app/enricher.py  (or similar path — check the project structure). It enriches company records by calling external sources (Crunchbase, Comparably, BuiltIn, Glassdoor) and writes results to the  jobs.companies  PostgreSQL table via SQLAlchemy ORM. The  Company  model is in  app/models.py  under the  jobs  schema.
Task
Add three guardrails to the  CompanyEnricher  class. Do NOT refactor or restructure the class — only add the guardrails described below. Keep all existing method signatures, logic, and external source calls exactly as they are.
Guardrail 1: Only set  enriched_at  when at least one source returns real data
In the main  enrich()  method:
	•	Wrap all external source calls in  asyncio.gather(..., return_exceptions=True)  if not already done
	•	Track which sources successfully returned data in a  sources_used  list
	•	If  sources_used  is empty: do NOT set  company.enriched_at , set  company.enrichment_source = [] , log a warning:  f"Enrichment returned no data for company: {company.slug}" 
	•	If  sources_used  is non-empty: set  company.enriched_at = datetime.utcnow() , set  company.enrichment_source = sources_used 
	•	Individual source exceptions must be caught and logged as warnings — must NOT crash other sources
Guardrail 2: Field-level validation before writing to the Company model
Create a new private method  _apply_validated(self, company,  dict) . Validation rules (discard silently if invalid):
	•	 funding_total_usd : int or float, must be > 0 and < 1,000,000,000,000
	•	 culture_score : parseable as float, must be 0.0–5.0
	•	 ceo_approval_pct : parseable as int, must be 0–100
	•	 work_life_score : parseable as float, must be 0.0–5.0
	•	 founded_year : parseable as int, must be 1800–current year
	•	 num_employees_range : must be exactly one of  "1-10" ,  "11-50" ,  "51-200" ,  "201-500" ,  "501-1000" ,  "1001-5000" ,  "5001-10000" ,  "10001+" 
	•	 salary_min_usd ,  salary_max_usd : int or float, must be > 0 and < 10,000,000
	•	All other fields: accept if non-None and non-empty string
Log debug for every discarded field:  f"Enricher validation rejected field '{field}' value '{value}' for company {company.slug}" 
Guardrail 3: Never overwrite existing non-null data with null or empty values
Inside  _apply_validated() , before setting any field:
	•	If incoming value is  None ,  "" , or  []  — skip, do not overwrite
	•	Enrichment is additive only — a failing source cannot erase previously good data
Additional Requirements
	•	Use  logging  standard library only — no new dependencies
	•	 logger = logging.getLogger(__name__)  at module level
	•	Validation failures →  logger.debug() , source exceptions →  logger.warning() , no data returned →  logger.warning() 
	•	Do not change any method signatures called from outside the class
	•	Do not change database session handling or commit logic
	•	Add a brief docstring to  _apply_validated() 
What NOT to Change
	•	External source fetch methods
	•	Company SQLAlchemy model
	•	Celery task definitions
	•	Database columns or migrations
	•	API endpoint logic
	•	Existing pip dependencies
Verification
	1.	 enrich()  uses  return_exceptions=True  in  asyncio.gather 
	2.	 enriched_at  is only set when  sources_used  is non-empty
	3.	 _apply_validated()  exists and is called wherever source data is written
	4.	No existing tests are broken
